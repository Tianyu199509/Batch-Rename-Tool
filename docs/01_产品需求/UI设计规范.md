# Batch Rename Tool UI 设计规范

## 1. 整体风格 (Aesthetics)
- **极简与现代化 (Minimalist & Modern)**：遵循高标准的美学设计，采用沉浸式的深色/浅色模式，大面积留白与清晰的层级。
- **色彩规范 (Color Palette)**：
  - Primary (主色调): 高级黑/白 (如 `#171717` / `#F5F5F5`)。
  - Background (背景): 极简纯粹的灰度背景 (`#0A0A0A` 或 `#FAFAFA`)。
  - Text (文本): 高对比度的纯黑或纯白，辅以低对比度的次要文本 (`#737373`)。

## 2. 界面布局 (Layout)
整体采用单窗口无边框（可选）或标准系统边框，采用**完全首尾冻结的弹性滚动布局**：

### 2.1 顶部操作区 (Top Action Bar - 冻结)
- 左侧：**[选择文件夹]** 按钮，以及当前选中路径的面包屑展示。
- 右侧：在选择文件夹按钮旁，当存在数据时，展示次级操作按钮 **[重置修改]**。
- 中间操作板：批量操作输入框及按钮组 (前缀、后缀、替换规则)。

### 2.2 底部表格区 (Data Grid Area - 滚动区)
- 占据窗口中间的核心弹性区域 (`flex-1 overflow-auto`)。
- 表头固定 (Sticky Header)。
- 表头的辅助按钮（复制、粘贴）始终保持高对比度常驻显示，不依赖鼠标悬停。

### 2.3 底部执行区 (Footer - 冻结)
- 在窗口最底部右侧，高亮的 **[执行重命名]** 按钮，始终固定在界面底部，不会随表格一起滚动。

### 2.4 全局反馈 (Global Toast)
- 固定在屏幕底部居中（如 `bottom-8`），黑色背景带高斯模糊，淡入淡出动画，提供精简文字状态提示。
